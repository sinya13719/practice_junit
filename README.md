## 【２章：ユニットテストは何のためにテストするのか】
### （要約）
- ソフトウェアテストの3つの特徴
  - 事前条件（テストデータ、環境など）による制約がある
  - 実際のソフトウェアのふるまいを記録する
  - 期待される結果との検証
- テスト技法  
  - ホワイト・ブラックボックステスト  
  - 同値クラスに対するテスト：ソフトウェアが同様の結果をもたらす値を同値クラスとしてグループ化し、グループが膨大である場合、代表値を選び、テストを行う
